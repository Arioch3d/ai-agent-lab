# Python Programming Best Practices & Coding Style Guide

## Code Style & Formatting

### General Principles

- Follow **PEP 8** (Python Enhancement Proposal 8) as the standard style guide
- Use **4 spaces** for indentation (never tabs)
- Maximum line length of **88 characters** (Black formatter standard)
- Use meaningful, descriptive variable and function names
- Keep functions and methods focused on a single responsibility

### Naming Conventions

```python
# Constants: UPPER_CASE
MAX_RETRIES = 3
API_TIMEOUT = 30

# Functions & variables: lowercase_with_underscores
def calculate_total_price(items):
    user_data = {}
    return sum(items)

# Classes: PascalCase
class UserController:
    pass

# Private methods/attributes: _leading_underscore
def _internal_helper():
    pass
```

### Imports

- Place imports at the top of the file
- Order: standard library → third-party → local imports
- Group imports and separate with blank lines

```python
import os
import sys
from datetime import datetime
from typing import List, Dict, Optional

import numpy as np
import pandas as pd

from my_module import my_function
```

## Code Organization & Structure

### Function & Method Design

- Keep functions small and focused (ideally < 20 lines)
- Use type hints for parameters and return values
- Always include docstrings

```python
def fetch_user_data(user_id: int) -> Dict[str, any]:
    """
    Retrieve user data from the database.

    Args:
        user_id: The unique identifier for the user

    Returns:
        Dictionary containing user information

    Raises:
        ValueError: If user_id is invalid
    """
    if user_id <= 0:
        raise ValueError("user_id must be positive")
    # implementation
```

### Class Design

- Use inheritance and composition appropriately
- Implement `__init__` for initialization
- Use properties for computed attributes
- Document class purpose in docstring

```python
class DataAnalyzer:
    """Analyzes and processes data sets."""

    def __init__(self, data_source: str):
        self.data_source = data_source
        self._cache = {}

    @property
    def is_valid(self) -> bool:
        """Check if data source is accessible."""
        return os.path.exists(self.data_source)
```

## Best Practices

### Error Handling

- Catch specific exceptions, not generic `Exception`
- Use try-except-else-finally appropriately
- Provide meaningful error messages

```python
try:
    result = perform_calculation(data)
except ValueError as e:
    logger.error(f"Invalid data: {e}")
    raise
except IOError as e:
    logger.warning(f"IO issue: {e}")
    result = DEFAULT_VALUE
else:
    logger.info("Calculation successful")
finally:
    cleanup_resources()
```

### Using Built-in Functions

- Leverage list comprehensions and generators
- Use `enumerate()` instead of range(len())
- Prefer `all()`, `any()`, `map()`, `filter()` where appropriate

```python
# Good
squared = [x**2 for x in numbers if x > 0]
for index, item in enumerate(items):
    process(index, item)

# Avoid
for i in range(len(items)):
    item = items[i]
```

### String Handling

- Use f-strings for formatting (Python 3.6+)
- Use `.format()` for complex formatting
- Avoid string concatenation in loops

```python
name = "Alice"
age = 30
message = f"Hello, {name}. You are {age} years old."

# Build strings efficiently
lines = ["header"]
for item in items:
    lines.append(f"  - {item}")
result = "\n".join(lines)
```

### Dictionary & List Operations

- Use `.get()` with defaults to avoid KeyError
- Use dictionary comprehensions for clarity
- Prefer unpacking over indexing when possible

```python
# Safe dictionary access
value = config.get("timeout", 30)

# Dictionary comprehension
squared_dict = {x: x**2 for x in range(10)}

# Unpacking
x, y, *rest = [1, 2, 3, 4, 5]
a, b = b, a  # Swap
```

## Testing

### Test Organization

- Write tests alongside your code
- Use descriptive test names: `test_<function>_<scenario>`
- Use assertions with meaningful messages

```python
def test_calculate_total_with_empty_list():
    result = calculate_total([])
    assert result == 0, "Empty list should return 0"

def test_fetch_user_raises_error_for_invalid_id():
    with pytest.raises(ValueError):
        fetch_user(-1)
```

## Performance Considerations

- Use generators for large datasets
- Avoid unnecessary list copies
- Cache repeated calculations with `functools.lru_cache`
- Profile before optimizing

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def expensive_calculation(n):
    return sum(range(n))

# Use generators for memory efficiency
def read_large_file(filepath):
    with open(filepath) as f:
        for line in f:
            yield line.strip()
```

## Common Pitfalls to Avoid

- ❌ **Mutable default arguments**: `def func(items=[]):`
- ✅ **Use**: `def func(items=None): items = items or []`

- ❌ **Not closing resources**: Files left open
- ✅ **Use**: `with open(file) as f: ...`

- ❌ **Bare except**: `except:`
- ✅ **Use**: `except SpecificException:`

- ❌ **Comparing with `is`**: `if x is True:`
- ✅ **Use**: `if x == True:` or `if x:` for booleans

- ❌ **Global variables**: Reduces testability
- ✅ **Pass as parameters** or use class attributes
